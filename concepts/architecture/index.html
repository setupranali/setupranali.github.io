    Authentication¶

    Every request requires an API key:

    # app/security.py
def require_api_key(request: Request) -> TenantContext:
    api_key = request.headers.get("X-API-Key")

    # Validate key
    key_config = validate_key(api_key)

    # Return tenant context
    return TenantContext(
        tenant=key_config.tenant,
        role=key_config.role
    )

    Authorization (RLS)¶

    Row-level security is injected automatically:

    # app/query_engine.py
def apply_rls(query: str, context: TenantContext, dataset: Dataset) -> str:
    if dataset.rls and context.tenant:
        rls_clause = f"{dataset.rls.tenant_column} = '{context.tenant}'"
        query = inject_where_clause(query, rls_clause)
    return query

    Query Engine¶

    Translates semantic requests to SQL using SQLGlot:

    SQLGlot Integration: - Parses and validates SQL queries - Converts between database dialects automatically - Builds queries programmatically (prevents SQL injection) - Applies RLS filters safely using AST manipulation

    Query Flow: 

    {                                  SQLGlot AST
  "dataset": "sales",      â†’         (parsed)

    Cache Layer¶

    Redis-based caching with deduplication:

    # Cache key includes query + tenant
cache_key = hash(query_params, tenant_id)

# Check cache
cached = redis.get(cache_key)
if cached:
    return cached  # Cache hit

# Execute query
result = execute_query(...)

# Store in cache
redis.setex(cache_key, ttl=300, value=result)

    Adapter Layer¶

    Database-agnostic abstraction:

    class BaseAdapter(ABC):
    @abstractmethod
    def connect(self) -> Any:
        """Establish connection"""

    @abstractmethod
    def execute_query(self, sql: str) -> Tuple[List, List[Dict]]:
        """Execute SQL and return (rows, columns)"""

    Data Flow¶

    Query Flow¶

    flowchart LR
    A[BI Tool] -->|1. Request| B[API Gateway]
    B -->|2. Auth| C{Valid?}
    C -->|No| D[401 Error]
    C -->|Yes| E[Apply RLS]
    E -->|3. Check| F{Cached?}
    F -->|Yes| G[Return Cache]
    F -->|No| H[Generate SQL]
    H -->|4. Execute| I[Database]
    I -->|5. Results| J[Cache + Return]

    Detailed Steps¶

    1. Request Received
    2. API key extracted from header

    3. Request parsed (JSON or OData)

    4. Authentication

    5. API key validated

    6. Tenant context created

    7. Authorization

    8. RLS rules looked up

    9. WHERE clause prepared

    10. Cache Check

    11. Cache key computed

    12. Redis lookup performed

    13. Query Generation

    14. Semantic model resolved
    15. SQL constructed

    16. RLS injected

    17. Execution

    18. Connection obtained
    19. Query executed

    20. Results fetched

    21. Response

    22. Results cached
    23. JSON returned
